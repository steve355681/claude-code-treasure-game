## Commands

```bash
npm install       # install dependencies
npm run dev       # start dev server at http://localhost:3000 (auto-opens browser)
npm run build     # production build to build/
```

There is no lint script, type-check script, or test suite configured.

## Architecture

This is a single-page React 18 + TypeScript + Vite app. The entire game lives in `src/App.tsx` — there is no routing and no state management library.

**Game logic** (`src/App.tsx`):
- 3 `Box` objects (`{ id, isOpen, hasTreasure }`) stored in `useState`
- One box is randomly assigned treasure on init/reset
- Opening a box gives +$100 (treasure) or -$50 (skeleton)
- Game ends when the treasure box is opened or all boxes are opened
- Audio playback uses the HTML `Audio` API with assets imported directly from `src/audios/`

**UI components** (`src/components/ui/`): shadcn/ui components built on Radix UI primitives with Tailwind CSS. Import them with the `@/components/ui/` alias. These are pre-generated — do not modify them unless fixing a specific bug.

**Figma component** (`src/components/figma/ImageWithFallback.tsx`): auto-generated by Figma-to-code tooling. The `vite.config.ts` includes versioned package aliases (e.g. `'lucide-react@0.487.0': 'lucide-react'`) specifically to support Figma's versioned import syntax.

**Path alias**: `@` resolves to `./src`.

**Assets**:
- `src/assets/` — PNG images for the chest states (closed, opened with treasure, opened with skeleton, key icon)
- `src/audios/` — MP3 sound effects for chest open events

**Styling**: Tailwind CSS via `src/styles/globals.css` and `src/index.css`. The amber color palette is the primary theme.
